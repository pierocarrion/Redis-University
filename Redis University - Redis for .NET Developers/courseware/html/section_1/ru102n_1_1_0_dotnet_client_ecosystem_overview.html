The client ecosystem for Redis is disparate and vibrant. With the exception of the hiredis client, the client ecosystem was initially built out by the larger Redis community. There are different levels of abstraction that Redis clients can provide to users. It's helpful if we try to group those together for a conversation about Redis clients. Let's think of them in these groups:

  • Minimal Protocol Clients: These clients basically handle connection, wrap the RESP protocol (the wire protocol for Redis) and little else. They're quite minimalist and intended to provide maximum control to the user.
  • General Command Level Clients: The clients that directly expose the Redis command interface and provide some parsing for the result structures bubbling up from it. These have a more or less 1:1 mapping between commands in Redis and commands in the client.
  • Higher Level Abstract Clients: These clients are intended more to provide a friendly user interface to developers and provide some meaningful abstraction above the Redis command interface and paradigms the users are used to.
  • Framework Integrations: These clients are written to interact directly with a particular framework, often times they implement some very particular functionality in a way native to the framework.

There are a plethora of clients of each of these breeds throughout the .NET community. The one that's had the most consequential impact however is undoubtedly StackExchange.Redis.

StackExchange.Redis was written by Marc Gravell in collaboration with Nick Craver as part of their work at StackExchange. Marc had written an older client called BookSleeve which had some fundamental issues that he sought to address in StackExchange.Redis.

The library falls somewhere between the second and third tier clients, while also leaving open a more or less minimalist ad-hoc API that we'll explore in later sections. It's best described as that second-tier General Command Level Client of Redis. And it has been adapted by the community as such.

Consequentially the .NET Redis community has largely been shaped by this project. All of the other third and fourth level libraries we'll be looking at later in this course have this Library as a dependency. Consequentially it's important for us to dive into some of the fundamentals of the library to see how it works, its benefits, and some of its limitations.

Redis Transactions

Transactions in Redis allow you to apply multiple commands in in isolation to Redis sequentially. StackExchange.Redis exposes the ITransaction which operationally is similar to the IBatch in that it allows you to dispatch a series of tasks, which you can then await after calling the Execute method to run the transaction.

Unlike IBatch however, all commands sent through a transaction are guaranteed to be executed sequentially. And you can apply some limited conditions to the transaction to allow it to self terminate if a watched key changes. In this section we'll learn how to work with Transactions in StackExchange.Redis.
